Configurar Esquema "fortisol"

Copia y pega este código en el SQL Editor de tu Supabase VPS. Esto creará el esquema fortisol y todas las tablas dentro de él. 

-- Crear esquema personalizado
CREATE SCHEMA IF NOT EXISTS fortisol;

-- Cambiar al esquema para que las tablas se creen allí
SET search_path TO fortisol;

-- ==========================================
-- 1. PERMISOS DE ESQUEMA Y TABLAS
-- ==========================================
GRANT USAGE ON SCHEMA fortisol TO anon;
GRANT USAGE ON SCHEMA fortisol TO authenticated;
GRANT USAGE ON SCHEMA fortisol TO service_role;

GRANT ALL ON ALL TABLES IN SCHEMA fortisol TO anon;
GRANT ALL ON ALL TABLES IN SCHEMA fortisol TO authenticated;
GRANT ALL ON ALL TABLES IN SCHEMA fortisol TO service_role;

GRANT ALL ON ALL SEQUENCES IN SCHEMA fortisol TO anon;
GRANT ALL ON ALL SEQUENCES IN SCHEMA fortisol TO authenticated;
GRANT ALL ON ALL SEQUENCES IN SCHEMA fortisol TO service_role;

ALTER DEFAULT PRIVILEGES IN SCHEMA fortisol GRANT ALL ON TABLES TO anon;
ALTER DEFAULT PRIVILEGES IN SCHEMA fortisol GRANT ALL ON TABLES TO authenticated;
ALTER DEFAULT PRIVILEGES IN SCHEMA fortisol GRANT ALL ON TABLES TO service_role;

-- Permitir que las roles puedan usar el esquema de storage
GRANT USAGE ON SCHEMA storage TO anon;
GRANT USAGE ON SCHEMA storage TO authenticated;
GRANT USAGE ON SCHEMA storage TO service_role;

GRANT ALL ON ALL TABLES IN SCHEMA storage TO anon;
GRANT ALL ON ALL TABLES IN SCHEMA storage TO authenticated;
GRANT ALL ON ALL TABLES IN SCHEMA storage TO service_role;

-- Dar permisos específicos a la tabla de objetos de storage
GRANT ALL ON storage.objects TO anon;
GRANT ALL ON storage.objects TO authenticated;
GRANT ALL ON storage.objects TO service_role;

GRANT ALL ON storage.buckets TO anon;
GRANT ALL ON storage.buckets TO authenticated;
GRANT ALL ON storage.buckets TO service_role;

-- ==========================================
-- 2. POLÍTICAS DE STORAGE (PARA LAS FOTOS)
-- ==========================================
-- Estas políticas permiten que cualquier usuario pueda subir y ver fotos en el bucket 'fortisol-assets'
INSERT INTO storage.buckets (id, name, public) 
VALUES ('fortisol-assets', 'fortisol-assets', true)
ON CONFLICT (id) DO NOTHING;

DROP POLICY IF EXISTS "Permitir Subida" ON storage.objects;
DROP POLICY IF EXISTS "Permitir Lectura" ON storage.objects;
DROP POLICY IF EXISTS "Permitir Borrado" ON storage.objects;
DROP POLICY IF EXISTS "Permitir Update" ON storage.objects;

CREATE POLICY "Permitir Subida" ON storage.objects FOR INSERT TO anon, authenticated WITH CHECK (bucket_id = 'fortisol-assets');
CREATE POLICY "Permitir Lectura" ON storage.objects FOR SELECT TO anon, authenticated USING (bucket_id = 'fortisol-assets');
CREATE POLICY "Permitir Borrado" ON storage.objects FOR DELETE TO anon, authenticated USING (bucket_id = 'fortisol-assets');
CREATE POLICY "Permitir Update" ON storage.objects FOR UPDATE TO anon, authenticated USING (bucket_id = 'fortisol-assets');

Modificar Código del Cliente

Para que el sistema sepa que debe buscar en el esquema fortisol en vez de public, yo modificaré el archivo src/lib/supabase.ts así:

export const supabase = createClient(supabaseUrl, supabaseAnonKey, {
  db: {
    schema: 'fortisol'
  }
});

¡Plan de Acción Maestro!

Como el Supabase viejo está bloqueado, haremos esto:

  1. Conecta esta app a tu VPS (cambia las variables en el menú Settings de la izquierda).
  2. Entra al panel /admin de esta web.
  3. Empieza a crear los productos y subir las fotos desde el formulario.
  4. La app se encargará de crear todo en el esquema fortisol y subir los archivos a MariaSuite automáticamente.
5

Configuración en Vercel

Como tu web está en Vercel, debes hacer esto para que los cambios se vean en internet:

  1. Entra a tu dashboard de Vercel.
  2. Ve a la pestaña Settings -> Environment Variables.
  3. Edita VITE_SUPABASE_URL con tu nueva URL de MariaSuite.
  4. Edita VITE_SUPABASE_ANON_KEY con tu nueva ANON KEY.
  5. Haz un Redeploy de tu rama de producción para que tome los cambios.
